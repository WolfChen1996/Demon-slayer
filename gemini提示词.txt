鬼灭之刃-呼吸法特效页面提示词：

创建一个网页，通过鼠标拖动或者手机端手指拖动，显示动漫鬼灭之刃中，水之呼吸、炎之呼吸、雷之呼吸的特效 (Demon Slayer Breathing Styles FX)
(纯 HTML/JS/CSS，基于 React)，支持手机和电脑（响应式）。
核心交互：
按住拖动：仅在按住鼠标或手指触摸屏幕并移动时产生特效。松开后特效停止（带惯性消散）。
拖动阻断保护：在画布上画线拖动经过按钮或菜单时，不应被打断（通过 Global Pointer Events 和 Pointer-Events 控制）。

三大呼吸特效 (Canvas 实现)
1. 水之呼吸 (Water Breathing)
视觉风格：深蓝色+天蓝色+白色高光的“丝带”状水流。参考动漫中的“水面斩”效果。
渲染技术：
Ribbon Mesh：根据鼠标路径生成多层丝带，并在尾部平滑变细（Tapering）。
Laplacian Smoothing：对路径点进行平滑处理，消除直线段的生硬感，变成流畅的曲线。
Batch Rendering：将所有粒子绘制合并为单一路径，极大优化性能。

粒子效果：
浪花 (Foam)：大量白色不规则圆形粒子在水流周围生成。
消散：浪花的消失速度比水流本体慢 ~0.3秒，形成滞留感。

2. 炎之呼吸 (Flame Breathing)
视觉风格：红/橙/黄/白多层叠加的火焰，头部圆润，尾部消散。
渲染技术：
Dragon Motion：火焰丝带通过正弦波（Sine Wave）模拟“蛇形”游走，看起来像一条火龙。
动态粗细：火焰宽度随距离产生剧烈的随机脉冲变化，模拟燃烧的不稳定性。
尾部处理：尾部逐渐变细并破碎成粒子消散，而不是直接消失。
粒子效果：
火星 (Embers)：大量红/黄色粒子四散飞舞。
残留：部分火星会停留在划过的路径上不动，模拟燃烧后的余烬。

3. 雷之呼吸 (Thunder Breathing)
视觉风格：金色+白色的高亮闪电，边缘锋利，瞬间产生。
渲染技术：
Fractal Lightning：使用递归分形算法生成锯齿状闪电路径。
动态抖动：闪电位置随时间快速微小抖动，模拟电流的不稳定性。
速度感：闪电产生和消失非常快。
粒子效果：
电火花：简单的圆形发光粒子跟随闪电爆开。

背景特效 (3D Magic Circles)
通用设计：所有背景魔法阵均为 六芒星 (Hexagram) 形状。
透视：统一采用 3D 地板透视（rotateX(75deg)），看起来像平铺在地面上。
位置：屏幕底部中央偏下。
独立粒子：背景中有独立的粒子从屏幕底部向上漂浮，不随魔法阵旋转。
拖入图片的时候，替换背景

各属性差异：
水：
颜色：皇家蓝 (#4789F6)。
背景粒子：蓝色/白色气泡，向上漂浮并淡出。
火：
颜色：红色/橙色 (#ef4444)。
动态：魔法阵整体有“呼吸灯”效果（Slow Pulse）。
背景粒子：红/橙色火星。
雷：
颜色：金黄色 (#fbbf24)。
动态：两道细长的闪电分别沿着六芒星的两个三角形边缘游走。两道闪电位置完全对称（180度相对），且在转角处会正确弯折。
背景粒子：黄色圆形光点。

UI 界面
1. 底部控制栏 (Control Bar)
按钮：三个风格化按钮（水、火、雷）。
外观：
颜色鲜艳明亮（使用 500/600 色阶）。
高度较矮（扁平化设计）。
带玻璃拟态（Backdrop blur）和发光边框。
文字：


2. 悬浮菜单 (Floating Menu)
位置：右下角，默认折叠，点击 + 号展开。
功能开关：
显示/隐藏按钮。
显示/隐藏魔法阵。
显示/隐藏中心提示文字。
中/English 语言切换。
添加上传图片的按钮，方便手机用户更换背景
重置背景（仅在有自定义背景时显示）。
BGM 控制：上一首、播放/暂停、下一首。


3. 中心提示
位置：屏幕顶部 1/4 处。
内容：多行文字，指导用户“按住拖动”、“拖入图片换背景”，提示右上角菜单位置。
性能优化：
使用 React.memo 避免背景不必要的重渲染。
Canvas 粒子使用 Batch Rendering 减少 Draw Calls。
限制最大粒子数量防止内存泄漏。
CSS 动画开启 GPU 加速 (will-change, transform-gpu)。
不要引入外部字体
引入外部js或者css库的时候，要引用在中国大陆可以访问的第三方托管平台

增加一点：
当用户第一次在屏幕拖动，触发特效的同时，从bgm1（无限城）的2分16秒开始播放；当用户第一次切换到炎之呼吸页面的时候，从bgm2（灶门炭治郎之歌）的1分33秒开始播放。除了这两个第一次，其他动作都不影响bgm，没有在菜单切歌或者暂停的话，bgm就一直单曲循环就行。
